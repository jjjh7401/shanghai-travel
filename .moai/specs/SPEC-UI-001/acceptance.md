## 1. 시스템 전역 요구사항 (Ubiquitous)

### AC-001: 한국어/중국어 병기 표시 [REQ-U-001]

```gherkin
Scenario: 장소명 한국어/중국어 병기
  Given 사용자가 앱의 어떤 페이지에 있을 때
  When 장소 이름이 표시되는 경우
  Then 한국어 이름이 주요 텍스트로 표시되고
  And 중국어 이름이 부제 또는 괄호 안에 함께 표시된다

  Examples:
    | 한국어명 | 중국어명 | 표시 형태 |
    | 점도덕 | 點都德 | "점도덕 (點都德)" |
    | 라오지우예 | 老九爷 | "라오지우예 (老九爷)" |
    | 와이탄 | 外滩 | "와이탄 (外滩)" |
```

### AC-002: 반응형 모바일 레이아웃 [REQ-U-002]

```gherkin
Scenario: 모바일 화면 최적화
  Given 사용자가 모바일 기기(화면 폭 320px ~ 430px)를 사용할 때
  When 앱의 모든 페이지를 탐색하면
  Then 가로 스크롤 없이 모든 콘텐츠가 표시되고
  And 터치 대상 요소의 최소 크기가 44x44px 이상이며
  And 텍스트가 읽기 어려울 정도로 작지 않다 (최소 14px)

Scenario: 태블릿/데스크톱 호환
  Given 사용자가 태블릿(768px) 또는 데스크톱(1024px+) 기기를 사용할 때
  When 앱에 접근하면
  Then 레이아웃이 적절히 확장되어 콘텐츠가 표시된다
```

### AC-003: 오프라인 데이터 접근 [REQ-U-003]

```gherkin
Scenario: 오프라인 상태에서 장소 정보 접근
  Given 사용자가 오프라인 데이터를 미리 다운로드했을 때
  And 네트워크 연결이 끊긴 상태일 때
  When 장소 목록 페이지에 접근하면
  Then 모든 장소의 이름, 카테고리, 설명이 표시되고
  And 각 장소의 추천 메뉴와 가격 정보가 표시된다

Scenario: 오프라인 상태에서 여행 팁 접근
  Given 사용자가 오프라인 데이터를 미리 다운로드했을 때
  And 네트워크 연결이 끊긴 상태일 때
  When 여행 팁 페이지에 접근하면
  Then 공항 이동 비교표, 결제 팁, 교통 정보가 모두 표시된다
```

### AC-004: 이중 통화 표시 [REQ-U-004]

```gherkin
Scenario: CNY/KRW 이중 통화 표시
  Given 앱에서 금액이 표시되는 모든 위치에서
  When 가격 정보가 렌더링될 때
  Then CNY(위안) 금액이 먼저 표시되고
  And KRW(원) 환산값이 함께 표시된다

  Examples:
    | CNY | KRW 환산 | 표시 형태 |
    | 94위안 | ~18,800원 | "94 CNY (~18,800 KRW)" |
    | 356위안 | ~71,200원 | "356 CNY (~71,200 KRW)" |
    | 2위안 | ~400원 | "2 CNY (~400 KRW)" |
```

## 2. 이벤트 기반 요구사항 (Event-Driven)

### AC-005: 일차별 필터링 [REQ-E-001]

```gherkin
Scenario: Day 1 탭 선택
  Given 사용자가 지도 페이지에 있을 때
  When Day 1 탭을 선택하면
  Then Day 1에 해당하는 장소 마커만 지도에 표시되고
  And Day 2, Day 3 장소 마커는 숨겨진다
  And Day 1 경로 폴리라인이 표시된다

Scenario: 전체 보기
  Given 사용자가 일차별 필터가 적용된 상태일 때
  When "전체" 탭을 선택하면
  Then 모든 일차의 장소 마커가 지도에 표시된다
```

### AC-006: 카테고리 필터 [REQ-E-002]

```gherkin
Scenario: 맛집 카테고리 필터
  Given 사용자가 지도 페이지에 있을 때
  When "맛집" 카테고리 필터를 선택하면
  Then restaurant 카테고리의 장소만 하이라이트되고
  And 다른 카테고리 마커는 흐리게(dimmed) 표시된다

Scenario: 복합 필터 (일차 + 카테고리)
  Given 사용자가 Day 2를 선택한 상태일 때
  When "카페" 카테고리 필터를 추가로 선택하면
  Then Day 2의 카페 장소만 하이라이트된다 (루이싱커피)
```

### AC-007: 장소 마커 상세 카드 [REQ-E-003]

```gherkin
Scenario: 마커 탭 시 상세 카드 표시
  Given 사용자가 지도 페이지에서 장소 마커를 볼 때
  When 점도덕(點都德) 마커를 탭하면
  Then 하단에 장소 상세 카드가 슬라이드 업으로 표시되고
  And 카드에 "점도덕 (點都德)" 이름이 표시되며
  And 카테고리 "맛집"이 표시되고
  And 추천 메뉴 "호미 창펑, 하가오"가 표시되며
  And "장소 상세 보기" 링크가 포함된다
```

### AC-008: 도보 경로 안내 [REQ-E-004]

```gherkin
Scenario: 두 장소 간 도보 경로 표시
  Given 사용자가 장소 상세 페이지에서 "다음 장소까지 경로" 버튼을 볼 때
  When 해당 버튼을 탭하면
  Then 지도에 현재 장소에서 다음 장소까지 도보 경로가 표시되고
  And 예상 도보 소요 시간(분)이 표시되며
  And 거리(km/m)가 함께 표시된다
```

### AC-009: WeChat 딥링크 [REQ-E-005]

```gherkin
Scenario: WeChat 예약 버튼 동작
  Given 사용자가 도운서점 장소 상세 페이지에 있을 때
  And 해당 장소에 WeChat 예약 정보가 있을 때
  When "WeChat으로 예약" 버튼을 탭하면
  Then WeChat 앱이 열리거나
  And WeChat 미설치 시 "WeChat을 설치하세요" 안내 메시지가 표시된다

Scenario: WeChat 예약 안내
  Given 사용자가 도운서점 장소 상세 페이지에 있을 때
  When 예약 정보 섹션을 확인하면
  Then "사전 위챗 예약 필수" 안내가 강조 표시되고
  And "무료 입장 (180위안 절약)" 팁이 표시되며
  And "1인 1메뉴 구매 필수" 조건이 표시된다
```

### AC-010: 따종 딥링크 [REQ-E-006]

```gherkin
Scenario: 따종 앱 딥링크
  Given 사용자가 젤네일 서비스 정보를 보고 있을 때

Scenario: 따종 할인 안내
  Given 사용자가 젤네일 관련 정보를 볼 때
  Then "따종 앱 첫 구매 할인 적용 가능" 팁이 표시되고
  And "약 150위안 (~30,000원)" 가격 안내가 표시된다
```

### AC-011: 오프라인 다운로드 [REQ-E-007]

```gherkin
Scenario: 오프라인 데이터 다운로드
  Given 사용자가 설정 페이지에 있을 때
  And 네트워크가 연결된 상태일 때
  When "오프라인 데이터 다운로드" 버튼을 탭하면
  Then 다운로드 진행률이 퍼센트로 표시되고
  And 다운로드 완료 시 "오프라인 사용 준비 완료" 메시지가 표시되며
  And 예상 용량(약 25MB)이 사전에 안내된다

Scenario: 다운로드 완료 상태 확인
  Given 오프라인 데이터 다운로드가 완료된 상태일 때
  When 설정 페이지에 접근하면
  Then "다운로드 완료" 상태가 표시되고
  And 캐시 크기 정보가 표시되며
  And "캐시 삭제" 옵션이 제공된다
```

### AC-012: 경고 강조 표시 [REQ-E-008]

```gherkin
Scenario: 위험 경고 강조
  Given 사용자가 라오지우예 장소 상세 페이지에 있을 때
  When 경고 정보가 표시될 때
  Then "백탕 잘못 고르면 개구리 탕이 될 수 있음! 꼭 확인" 경고가
  And 빨간색 배경 또는 경고 아이콘과 함께 시각적으로 강조되어 표시된다

Scenario: 주의 팁 표시
  Given 사용자가 장소 상세 페이지에서 팁 섹션을 볼 때
  When severity가 "caution" 또는 "danger"인 팁이 있으면
  Then 해당 팁이 일반 팁보다 시각적으로 구분되어 표시된다
  And 경고 아이콘이 팁 옆에 표시된다
```

## 3. 상태 기반 요구사항 (State-Driven)

### AC-013: 오프라인 지도 렌더링 [REQ-S-001]

```gherkin
Scenario: 오프라인 지도 표시
  Given 사용자가 오프라인 데이터를 다운로드한 상태이고
  And 네트워크가 끊긴 상태일 때
  When 지도 페이지에 접근하면
  Then 캐싱된 지도 타일로 상하이 중심부 지도가 표시되고
  And 장소 마커가 정상적으로 표시되며
  And "오프라인 모드" 인디케이터가 화면 상단에 표시된다

Scenario: 오프라인 시 경로 탐색 제한
  Given 네트워크가 끊긴 상태일 때
  When 도보 경로 탐색을 시도하면
  Then "오프라인 상태에서는 실시간 경로 탐색이 불가합니다" 안내가 표시되고
  And 사전 계산된 경로 정보(소요 시간, 거리)는 표시된다
```

### AC-014: AliPay 결제 경고 [REQ-S-002]

```gherkin
Scenario: 200위안 초과 결제 경고
  Given 장소의 비용 추정이 200위안을 초과할 때
  When 해당 장소의 비용 정보가 표시되면
  Then "200위안 초과 시 AliPay 3% 수수료 발생" 경고 배너가 표시되고
  And "분할 결제 권장: 200위안 미만으로 나누어 결제하세요" 안내가 함께 표시된다

  Examples:
    | 장소 | 금액 | 경고 표시 |
    | 라오지우예 | 356위안 | 표시됨 |
    | 리바이시에 | 202위안 | 표시됨 |
    | 동북사계교자왕 | 94위안 | 미표시 |
    | 샤오양셩지엔 | 104위안 | 미표시 |
```

### AC-015: 시간대 기반 야경 추천 [REQ-S-003]

```gherkin
Scenario: 저녁 시간 야경 스팟 우선 추천
  Given 현재 시각이 18:00 이후일 때
  When 사용자가 홈 페이지 또는 지도 페이지에 접근하면
  Then "지금 가기 좋은 야경 스팟" 섹션이 상단에 표시되고
  And 북와이탄, 황푸공원, 와이탄이 추천 목록에 포함된다
  And "북와이탄이 와이탄보다 사람이 적고 뷰가 좋습니다" 팁이 표시된다

Scenario: 낮 시간 일반 추천
  Given 현재 시각이 18:00 이전일 때
  When 사용자가 홈 페이지에 접근하면
  Then 야경 추천 섹션이 표시되지 않고
  And 현재 일차의 일반 코스가 표시된다
```

### AC-016: 근접 장소 알림 [REQ-S-004]

```gherkin
Scenario: 근접 장소 자동 알림
  Given 사용자가 위치 권한을 허용한 상태이고

  And 카드에 장소명, 카테고리, 주요 추천 메뉴가 포함되며
  And "길 찾기" 버튼이 포함된다

Scenario: 위치 권한 미허용
  Given 사용자가 위치 권한을 거부한 상태일 때
  Then 근접 알림 기능이 비활성화되고
  And 기능 사용을 위한 위치 권한 안내가 설정 페이지에 표시된다
```

## 4. 금지 사항 (Unwanted)

### AC-017: Google Maps 미사용 [REQ-N-001]

```gherkin
Scenario: Google Maps API 미사용 확인
  Given 앱의 전체 소스 코드에서
  When 지도 관련 의존성을 검사하면
  Then Google Maps API (maps.googleapis.com)에 대한 참조가 없어야 하고
  And @googlemaps 패키지에 대한 의존성이 없어야 한다
```

### AC-018: 개인정보 미전송 [REQ-N-002]

```gherkin
Scenario: 위치 데이터 외부 전송 금지
  Given 사용자가 위치 권한을 허용한 상태일 때
  When 앱이 사용자의 위치 정보를 수집하면
  Then 위치 데이터는 클라이언트 측에서만 처리되고
  And 외부 서버(자체 서버 포함)로 전송되지 않는다
  And 위치 기록이 LocalStorage에도 저장되지 않는다

Scenario: 검색/필터 데이터 외부 전송 금지
  Given 사용자가 카테고리 필터를 사용할 때
  When 필터 동작이 실행되면
  Then 사용자의 필터 선택 정보가 외부 서버로 전송되지 않는다
```

### AC-019: 비로그인 접근 [REQ-N-003]

```gherkin
Scenario: 로그인 없이 핵심 기능 사용
```

## 5. 선택 기능 인수 기준 (Optional)

### AC-020: 실시간 환율 [REQ-O-001]

```gherkin
Scenario: 실시간 환율 반영
  Given 네트워크가 연결된 상태이고
  And 환율 API가 정상 작동할 때

  Then 실시간 CNY-KRW 환율이 적용된 KRW 금액이 표시되고
  And 환율 기준 시각이 작은 텍스트로 표시된다

  Given 환율 API에 접근할 수 없을 때
  When 금액이 표시되면
  Then 기본 환율(1 CNY = 200 KRW 등 고정값)이 적용되고
  And "기준 환율로 계산된 금액입니다" 안내가 표시된다
```

### AC-021: 방문 체크 및 진행률 [REQ-O-002]

```gherkin
Scenario: 장소 방문 체크
  Given 사용자가 장소 목록 페이지에 있을 때

  And 체크 상태가 LocalStorage에 저장된다
```
